# Правила проекта KobraS1BotWiki

Телеграм-бот поддержки для групп по 3D-принтерам Anycubic (Kobra S1 и др.): читает вопросы в чате, ищет по вики, отвечает ссылкой. Публичный репозиторий.

## Ручные ответы (manual_qa) — приоритет

- Когда в `data/missed_questions.json` или `data/bad_answers.json` всплывает реальный вопрос, на который в вики нет хорошей страницы, **по возможности добавляй ручной ответ** в `data/manual_qa.json`, а не просто отсекай.
- Ответ должен быть **кратким и понятным** — суть сразу, без воды, чтобы пользователю не пришлось никуда переходить.
- **Прикладывай ссылку(и) на официальные источники Anycubic** (`https://wiki.anycubic.com/...`), если есть подходящая страница — как дополнение к тексту, а не вместо него. Ссылку давай только проверенную; не выдумывай URL. Формат — как в записи «Калибровка стола и первый слой»: короткий текст, затем строки со ссылками.
- **Если точной официальной ссылки Anycubic нет — отвечай только текстом.** Не приклеивай общий/неточный URL (например, общий filament-guide к узкой теме) лишь бы была ссылка.
- Структура записи: `{keys, title, answer, ts}`. `keys` — нормализованные подстроки-триггеры (длиной ≥ 6 символов), которые встречаются в вопросах; добавляй несколько формулировок. Проверяй матчинг через `app.bot.manual_qa.find_manual_qa_answer` перед коммитом.

## Очереди разбора (их обрабатываю я, Claude)

- `missed_questions.json` (низкий score, бот молчал) и `bad_answers.json` (отмечены через панель) — это аналитика пробелов, а не архив. Для каждой записи: понять причину → поправить эвристику в `app/bot/heuristics/` **или** добавить manual_qa **или** признать оффтоп. **После разбора записи удалять** (очищать файл) и пушить.
- «Последние ответы бота» (`.cache/recent_replies.json`) — прокручиваемая лента; разбираю так же (правлю эвристики), затем чищу кнопкой «Очистить ленту» в панели (или `POST /replies/clear`).
- Болтовню/фрагменты/оффтоп отсекаю узкими функциями-эвристиками в `app/bot/heuristics/_banter.py`, регистрирую в `_filter.py::_is_non_wiki_chatter_message`, реэкспортирую в `__init__.py` и `text_heuristics.py`, и на каждую пишу тест в `tests/test_bad_answers_chatter.py` (обязательно с негативными кейсами — реальные вопросы не должны блокироваться).

## Деплой и сервер

- Бот запускается **только** под пользователем `anycubicwikibot` (git-алиас `github-kobra` настроен только у него; под root `git pull` падает и плодит дубли). Подробности — в `.cursor/memories.md` (секция `server-deployment`; файл локальный, в git не коммитится).
- Поток: запушить с локалки → на сервере `git pull --ff-only` под `anycubicwikibot` → `sudo systemctl restart kobras1botwiki.service` (суффикс `.service` обязателен для NOPASSWD-sudo).
- Перед своим push всегда `git pull` — живой бот сам коммитит data-файлы. data-файлы на сервере перед pull сбрасывать (`git checkout -- data/`).

## Безопасность

- `TELEGRAM_BOT_TOKEN` и `.env` никогда не коммитить и не показывать в панели.
- Репозиторий публичный: перед записью в `missed_questions.json` работает санитайзер (ссылки/контакты → плейсхолдеры) + фильтр запрещённых/политических тем в `add_missed_question`. Фильтр держим консервативным — без двусмысленных слов.
