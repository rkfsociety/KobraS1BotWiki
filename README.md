# Kobra S1 Bot Wiki

Бот читает сообщения в Telegram (группы, супергруппы, личные чаты), пытается найти релевантную страницу в вашей вики (по `WIKI_SITEMAP_URL`, обычно это `sitemap.xml` на сайте вики) и, если совпадение достаточно хорошее, отвечает ссылкой на страницу.

## Структура кода

Логика бота разнесена по пакету `app/bot/` (точка входа — `python -m app.bot`, см. `app/bot/__main__.py` и `app/bot/lifecycle.py`):

| Модуль | Роль |
|--------|------|
| `constants.py` | константы, пути к локальным JSON в `.cache/` |
| `i18n.py` | язык ответа (ru/en), строки интерфейса |
| `stores.py` | кэш ответов, feedback после `/error`, фиксы ссылок `/fix` |
| `text_heuristics.py` | эвристики: модель в тексте, тема, код ошибки |
| `wiki_ranking.py` | поиск по индексу с бонусами/штрафами по URL и модели |
| `error_codes_wiki.py` | выбор страницы `/error-codes/...` |
| `error_display.py` | карточки кодов ошибок и перевод |
| `design_replies.py` | короткие ответы из справочника конструкции (без вики) |
| `clarify.py` | уточнение модели и цепочки reply |
| `handlers.py` | команды и обработка сообщений |
| `admin_access.py` | кто считается администратором для служебных команд |
| `help_text.py` | текст команды `/help` (plain text, ru/en; отдельно для админа и участника) |
| `ephemeral.py` | автоудаление пары «команда + ответ бота» в группах (не в личке и не в чатах из `ephemeral_exempt_chat_ids`) |
| `ops_notify.py` | служебные уведомления в чат (старт, ошибки, перезапуски; см. `OPS_NOTIFY_CHAT_ID`) |
| `lifecycle.py` | логирование, `Application`, индексация, polling |

В корне пакета `app/` также: **`config.py`**, **`default_developers.py`**, **`default_ephemeral_exempt.py`**, **`default_ops_chat.py`** (дефолтный `chat_id` для служебных уведомлений; см. `OPS_NOTIFY_CHAT_ID`).

В корне репозитория лежит **`sitemap.xml`** — снимок карты сайта вики (для справки и офлайн-просмотра). Рабочий URL по-прежнему задаётся в `.env` как `WIKI_SITEMAP_URL` (часто это HTTPS-адрес того же sitemap на сервере вики).

Каталог **`logs/`** — ротируемый файл **`logs/bot.log`** (решения бота, индексация, ошибки). Каталог **`.cache/`** создаётся при работе бота (индекс, состояние, локальные сторы) и **не коммитится** (см. `.gitignore`).

## Что нужно

- Python 3.11+ (желательно)
- Токен Telegram-бота из BotFather

## Быстрый старт

1) Установить зависимости:

```bash
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```

2) Создать `.env` рядом с `requirements.txt` (можно скопировать из примера):

```bash
copy .env.example .env
notepad .env
```

Важно: файл `.env` **не коммитьте** — он добавлен в `.gitignore`. В репозиторий коммитится только `.env.example`.

Минимально нужно заполнить:

- `TELEGRAM_BOT_TOKEN`
- `WIKI_BASE_URL`
- `WIKI_SITEMAP_URL`

3) Запустить:

```bash
python -m app.bot
```

Или (Windows) открыть отдельное окно с логами:

```bat
start-bot.cmd
```

Или (Linux/macOS):

```bash
./start-bot.sh
```

Остановить (если запускали через `start-bot.cmd`):

```bat
stop-bot.cmd
```

Перезапустить:

```bat
restart-bot.cmd
```

На Linux: `./stop-bot.sh`, `./restart-bot.sh`.

4) Добавить бота в группу и написать в группе команду:

`/id`

Бот ответит числовым `chat_id`. Если нужно ограничить бота только выбранными чатами или темами форума — см. `ALLOWED_CHAT_IDS` и `ALLOWED_TOPIC_IDS` в `.env.example` и команду `/status` (показывает, разрешён ли текущий чат/тема).

## Команды бота

- `/help` — справка **обычным текстом** (без HTML): для **администраторов чата** — полный список служебных команд; для **участников** — как задавать вопросы и что служебные команды им недоступны. В **личке** служебные команды доступны.

Служебные команды ниже в **группах и супергруппах** обрабатываются только если вызывающий — **администратор или владелец** этого чата, либо user_id из списка **разработчиков** (`app/default_developers.py` и `DEVELOPER_USER_IDS`); иначе бот **не отвечает** на команду (молча игнорирует). В личке с ботом служебные команды доступны.

- `/id` — ID чата и тип чата
- `/admincheck` — в группе: строка `get_chat_member` и флаги «разработчик» / «доступ к служебным командам» у бота
- `/wiki <запрос>` — поиск по вики по запросу
- `/ping`, `/status` — диагностика
- `/error` (reply на сообщение бота) — плохой ответ: перепоиск и обучение «не эта ссылка»
- `/fix <url>` (reply на сообщение бота) — указать правильную ссылку для того же запроса
- `/qaadd`, `/qalist`, `/qadel` — ручные пары вопрос–ответ (`data/manual_qa.json`; см. `/help`)
- `/update` — подтянуть код с GitHub и перезапустить бота (только админы чата / личка; нужны права на `git` в каталоге клона)

## Как это работает

- Из `WIKI_SITEMAP_URL` берётся список страниц
- Страницы скачиваются и складываются в кэш (`CACHE_PATH`, см. `.env.example`)
- Для каждой страницы строится «текст документа» (title + текст страницы без скриптов/стилей)
- Для каждого сообщения в чате считается похожесть (RapidFuzz `token_set_ratio`), плюс эвристики по модели и URL
- Если лучший результат \(\ge\) `MIN_SCORE`, бот отвечает ссылкой (с опциональными уточнениями модели и защитой от нерелевантных URL)

## Настройки

Смотрите `.env.example`.

Полезное:

- `MIN_SCORE`: порог (0..100). Если слишком много ложных — поднимите.
- `QUESTIONS_ONLY=true`: отвечать только на вопросы («?» или вопросительные слова).
- `ALLOWED_CHAT_IDS` / `ALLOWED_TOPIC_IDS`: ограничить работу конкретными группами и темами (см. комментарии в `.env.example`).
- `DEVELOPER_USER_IDS`: дополнительные user_id разработчиков (к базовому списку в `app/default_developers.py`): в группах — служебные команды как у админа чата, без антиспама и кулдаунов clarify.
- `EPHEMERAL_EXEMPT_CHAT_IDS`: дополнительные `chat_id`, где не автоудалять пару «команда + ответ» (к списку в `app/default_ephemeral_exempt.py`).
- `OPS_NOTIFY_CHAT_ID`: служебный чат для старта, ошибок и перезапусков (`0`/`off` — выкл.; по умолчанию см. `app/default_ops_chat.py`).
- `WIKI_MAX_PAGES`: ограничить количество страниц для индекса (чтобы первый запуск был быстрее).
- `MEMORY_LIMIT_MB` (Linux/macOS): жёсткий потолок виртуальной памяти процесса (`RLIMIT_AS`), например `512`. `0` — без лимита. При OOM из‑за малого лимита уменьшите индекс или поднимите значение.
- `REQUIRE_TRIGGER`: в группах для **не-вопросов** требовать @бота или ожидаемый reply; на сообщения, похожие на вопрос (`QUESTIONS_ONLY`), бот отвечает без @.

| `ops_notify.py` | отправка в служебный чат: старт, ошибки, перезапуски (`OPS_NOTIFY_CHAT_ID`) |

## Служебный чат (ошибки, перезапуски, старт)

В Telegram-канал **AnycubicWikiBotLog** с id **`-1003826125815`** (по умолчанию, см. `app/default_ops_chat.py`) бот шлёт:

- **зеркало консоли** — всё, что пишется в лог (INFO и выше), пачками каждые ~2 с (`OPS_LOG_MIRROR_ENABLED`, по умолчанию вкл. если задан `OPS_NOTIFY_CHAT_ID`);
- **старт** после `post_init` (username, число документов в индексе, pid);
- **необработанные исключения** в обработчиках (`on_error`);
- **перезапуск** перед остановкой polling после `/update` или фонового git autopull;
- **сбой `/update`** и **исключение** в фоновом `git autopull`.

Выключить зеркало консоли: `OPS_LOG_MIRROR_ENABLED=0`. Уровень: `OPS_LOG_MIRROR_LEVEL=DEBUG`. Интервал слива: `OPS_LOG_MIRROR_INTERVAL_SECONDS=2`.

Отключить или сменить чат: в `.env` переменная **`OPS_NOTIFY_CHAT_ID`** (`0` / `off` — выключить; иначе числовой `chat_id`).

## Логи на сервере (systemd + screen)

Если бот запущен через **`ensure-bot.sh`** в **screen**, вывод **`python -m app.bot`** попадает в **screen** и в файл **`logs/bot.log`**, а не в **`journalctl -u …`**. В журнале systemd обычно видны только строки скриптов при старте unit’а.

Смотреть лог бота:

```bash
sudo -u <пользователь_бота> tail -f /путь/к/KobraS1BotWiki/logs/bot.log
```

При **`LOG_DECISIONS=true`** в логе есть строки **`seen chat=`** и **`skip … reason=`** — по ним видно, почему бот не ответил на сообщение.

## Git и команда `/update`

Команда **`/update`** выполняет `git fetch` / синхронизацию с remote от **пользователя процесса бота**. Если репозиторий или **`.git/objects`** когда‑то создавались от **root**, возможна ошибка *insufficient permission for adding an object*. Исправление: выдать владельцем каталог клона пользователя бота, например `chown -R user:user /path/to/KobraS1BotWiki`.
