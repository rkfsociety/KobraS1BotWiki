# Конфигурация (.env)

Все параметры задаются в файле `.env` рядом с `requirements.txt`. Шаблон — `.env.example`.

## Обязательные

| Переменная | Описание |
|-----------|---------|
| `TELEGRAM_BOT_TOKEN` | Токен бота из @BotFather. **Никогда не коммитить.** |
| `WIKI_BASE_URL` | Базовый URL вики (например `https://wiki.anycubic.com`) |
| `WIKI_SITEMAP_URL` | URL sitemap.xml вики |

## Поиск и пороги

| Переменная | По умолчанию | Описание |
|-----------|-------------|---------|
| `MIN_SCORE` | `62` | Порог совпадения (0–100). Много ложных — поднять. |
| `CLARIFY_MIN_SCORE` | `50` | Порог для уточняющего вопроса о модели принтера. |
| `CLARIFY_ENABLED` | `true` | Задавать уточняющий вопрос при «почти уверенном» score. |
| `QUESTIONS_ONLY` | `false` | Отвечать только на вопросы (с «?» или вопросительными словами). |
| `REQUIRE_TRIGGER` | `false` | Для не-вопросов требовать @бота или reply на бота. |
| `RU_LAYER_ENABLED` | `true` | Расширять русские запросы английскими ключевыми словами. |

## Доступ и чаты

| Переменная | По умолчанию | Описание |
|-----------|-------------|---------|
| `ALLOWED_CHAT_IDS` | _(все)_ | Где бот может отвечать: список chat_id через запятую. Пусто = отвечать во всех чатах. |
| `ALLOWED_TOPIC_IDS` | _(все)_ | Где бот может отвечать: список topic_id тем форума (пусто = все темы разрешённых чатов). |
| `REQUIRE_CAN_REPLY` | `false` | Проверять `getChatMember` перед обработкой. |
| `DEVELOPER_USER_IDS` | — | Дополнительные user_id разработчиков: служебные команды как у админа. |
| `EPHEMERAL_EXEMPT_CHAT_IDS` | — | Чаты, где не автоудалять пару «команда + ответ». |

## Поведение в группах

Бот собирает входящую статистику и текстовые вопросы из всех групп и тем, сообщения которых Telegram ему доставляет, независимо от `ALLOWED_*`. Эти списки ограничивают только исходящие ответы: вне разрешённого чата/темы бот молчит, но вопрос попадает в общую очередь `data/missed_questions.json` для последующего разбора.

| Переменная | По умолчанию | Описание |
|-----------|-------------|---------|
| `REPLY_REVIEW_MENTION` | `off` | Username (без @) — добавлять в конце ответа для уведомления. |
| `REACTION_LOG_ADMIN_ONLY` | `1` | Логировать реакции 💩/👎 только от админов. |
| `NEGATIVE_REACTION_EMOJIS` | `💩 👎` | Набор «плохих» эмодзи для логирования. |

## Индексация вики

| Переменная | По умолчанию | Описание |
|-----------|-------------|---------|
| `WIKI_MAX_PAGES` | _(без лимита)_ | Ограничить число страниц в индексе (быстрее первый запуск). |
| `INDEX_BATCH_SIZE` | `10` | Страниц за один пакет индексации. |
| `INDEX_INTERVAL_SECONDS` | `5` | Пауза между пакетами. |

## Логи и диагностика

| Переменная | По умолчанию | Описание |
|-----------|-------------|---------|
| `LOG_DECISIONS` | `false` | Писать в `logs/bot.log` строки `seen`/`skip reason=` — почему бот промолчал. |
| `MEMORY_LIMIT_MB` | `0` | Потолок виртуальной памяти (Linux/macOS, `RLIMIT_AS`). `0` = без лимита. |

## Служебный чат

| Переменная | По умолчанию | Описание |
|-----------|-------------|---------|
| `OPS_NOTIFY_CHAT_ID` | `-1003826125815` | chat_id канала для уведомлений. `0`/`off` = выключить. |
| `OPS_LOG_MIRROR_ENABLED` | `true` | Зеркало ответов бота в служебный чат. |
| `OPS_LOG_MIRROR_LEVEL` | `INFO` | Уровень логов для зеркала. |
| `OPS_LOG_MIRROR_INTERVAL_SECONDS` | `2` | Интервал сброса буфера зеркала. |

## Git и автообновление

| Переменная | По умолчанию | Описание |
|-----------|-------------|---------|
| `GIT_AUTOPULL_ENABLED` | `false` | Периодический `git pull` в фоне (без ручного `/update`). |
| `MANUAL_QA_GIT_PUSH` | `false` | Коммитить и пушить `data/manual_qa.json` после изменений через панель. |

## Фильтр болтовни

Бот автоматически молчит на нетехнический чат (реализовано в `app/bot/text_heuristics.py`):

- Сравнения принтеров, обсуждение покупки б/у, планы печати
- Статусы («запускаю первый слой», «запустился в многоцвет»)
- Сарказм, риторические вопросы, мнения в треде
- Числа/символы без слов («35?», «40%?»)
- Короткие анафорические вопросы без 3D-контекста («А зачем он?»)
- Болтовня о ПК, алкоголе, сне — вне темы печати
- Запросы мастера по ремонту в городе, соцсети

При `LOG_DECISIONS=true` причина молчания видна в `logs/bot.log` как `skip reason=conversational_chatter`.
