# Команда /app для группы — план реализации

> **Для агентских исполнителей:** использовать TDD; выполнять шаги последовательно.

**Цель:** добавить служебную команду `/app`, которая публикует в группе кнопку перехода в личный чат бота для запуска Mini App.

**Архитектура:** новый обработчик в `app/bot/handlers/_cmd_basic.py` использует существующую проверку прав админа и значение `bot_username` из `application.bot_data`. Команда регистрируется в основном polling-приложении и экспортируется через пакет обработчиков. Ссылка имеет формат `https://t.me/<bot_username>?start=app`; существующий `cmd_start` остаётся без изменений.

**Технологии:** Python 3, python-telegram-bot, pytest.

## Ограничения

- Не создавать новую git-ветку.
- Не включать в коммит уже изменённые файлы очереди и эвристик.
- В группе команду могут запускать только администраторы.
- После проверки: commit, push, `git pull --ff-only` перед push по правилам проекта, затем restart systemd-сервиса.

## Шаги

1. Добавить тест обработчика, проверяющий текст ответа и URL inline-кнопки; убедиться, что тест падает из-за отсутствующей команды.
2. Реализовать `cmd_app`, экспортировать его и зарегистрировать `CommandHandler("app", cmd_app)`.
3. Запустить тест обработчика и полный набор тестов.
4. Проверить diff и состояние рабочей копии, добавить только файлы этой задачи и сделать commit.
5. Выполнить `git pull --ff-only`, push текущей ветки, затем на сервере выполнить pull и restart `kobras1botwiki.service`.
