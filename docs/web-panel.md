# Веб-панель администратора

Встроенная панель (`app/web_panel.py`) запускается в фоновом потоке внутри процесса бота — отдельный сервис не нужен. Включается при `PANEL_ENABLED=true` или автоматически, если задан `PANEL_PASSWORD`.

## Возможности

- **Дашборд** — лента последних ответов бота с кнопкой «Отметить ошибочным» (⚑), статус, размер индекса, ключевые параметры
- **Ошибочные ответы** — список помеченных ответов (`data/bad_answers.json`); кнопка «Обработано» удаляет запись
- **Ручные Q&A** — просмотр / добавление / редактирование / удаление `data/manual_qa.json` (аналог `/qaadd`, `/qalist`, `/qadel`); правки сразу применяются без перезапуска
- **Фиксы ссылок** — управление переопределениями (`fixes.json`, аналог `/fix`)
- **Логи** — хвост `logs/bot.log` с фильтром по подстроке
- **Обновление** — «Проверить обновления» (git fetch) и «Обновить» (git pull + перезапуск)
- **Настройки** — редактирование `.env` прямо из браузера (кроме токена бота)

## Telegram Mini App

Админская версия Mini App открывается по адресу `/app`, например:
`https://kobras1bot.duckdns.org/app`.

Это одно приложение для одной группы. При открытии Telegram `initData` проверяется на сервере, затем бот через `get_chat_member` проверяет статус пользователя в `PANEL_ADMIN_CHAT_ID`. Доступ получают только `creator` и `administrator`.

В Mini App доступны дашборд, статистика, поиск по вики и очередь вопросов без ответа. Администратор может добавить ручной ответ или отметить вопрос оффтопом — изменения сразу попадают в общие данные бота.

### Пользовательский режим и история чата

Кнопка **«Режим пользователя»** в админском Mini App открывает предпросмотр того, что видит обычный участник. В этом режиме можно отправить вопрос боту и проверить пользовательский сценарий; админский дашборд и очередь при этом скрыты. Администратор может вернуться кнопкой **«Режим админа»**.

История Mini App хранится локально в базе SQLite `data/chat.sqlite3`. Для каждого пользователя записи разделяются по его Telegram ID, поэтому запрос истории возвращает только персональную переписку этого пользователя. База и её WAL/SHM-файлы не должны попадать в git.

Endpoint’ы пользовательского чата:

- `GET /api/app/chat/history` — возвращает историю текущего пользователя; поддерживает `limit` (от 1 до 50) и необязательный `before_id` для подгрузки более старых сообщений.
- `POST /api/app/chat/message` — принимает поле формы `text`, сохраняет вопрос и ответ бота и возвращает пару сообщений. Для обоих endpoint’ов требуется Bearer-сессия Mini App.

Ограничения отправки применяются отдельно для каждого Telegram ID: не более одного сообщения за 3 секунды и не более 20 сообщений за 10 минут. Повтор того же текста с уже сохранённым ответом в течение 10 секунд считается duplicate и возвращает существующую пару без повторной отправки запроса боту.

Если ручной ответ или достаточно точное совпадение вики не найдены, бот возвращает короткий fallback о невозможности ответить, сохраняет этот ответ в истории с источником `missing` и добавляет исходный вопрос в очередь `missed_questions`. Администратор разбирает очередь через `GET /api/app/missed`, а результат добавляет ручной ответ или отмечает вопрос оффтопом.

Обычный участник группы может получить пользовательскую Mini App-сессию, но не имеет доступа к админским данным: запросы к `/api/app/dashboard`, `/api/app/missed` и операциям очереди требуют роли администратора группы и для обычного участника возвращают `403`. Таким образом, админский dashboard и endpoint’ы `missed` не видны и недоступны обычному участнику.

Чтобы `/start` показывал кнопку открытия приложения, задайте:

```dotenv
PANEL_WEBAPP_URL=https://kobras1bot.duckdns.org/app
```

## Вход

### Через Telegram-бота (рекомендуется)

Не требует домена — работает на «голом» IP:порт.

1. На странице входа нажать «Войти через Telegram-бота»
2. Откроется `t.me/<бот>?start=<код>` — нажать **Start**
3. Бот через `get_chat_member` проверяет, что пользователь — **админ группы** `PANEL_ADMIN_CHAT_ID`
4. Код одноразовый, живёт 5 минут, привязан к браузеру

### По логину/паролю (запасной)

Активен только если задан `PANEL_PASSWORD`. Членство в группе не проверяется.

### Login Widget (опционально)

Endpoint `/tg-auth` поддерживает классическую кнопку «Log in with Telegram». Требует домен — в @BotFather → `/setdomain`. На «голом» IP не работает.

## Настройки панели

| Переменная | По умолчанию | Описание |
|-----------|-------------|---------|
| `PANEL_ENABLED` | `true` если задан пароль | Включить панель. Для TG-входа без пароля задайте явно `true`. |
| `PANEL_TG_LOGIN` | `true` | Вход через Telegram. |
| `PANEL_ADMIN_CHAT_ID` | `-1003881305021` | Группа, чьи админы могут входить. `0`/`off` — выключить TG-вход. |
| `PANEL_PASSWORD` | — | Пароль запасного входа. Пусто = только через Telegram. |
| `PANEL_USERNAME` | `admin` | Логин запасного входа. |
| `PANEL_HOST` | `0.0.0.0` | Адрес прослушивания. `127.0.0.1` — только локально. |
| `PANEL_PORT` | `8080` | Порт. |
| `PANEL_SESSION_TTL_SECONDS` | `86400` | Срок жизни сессии (секунды). |
| `PANEL_WEBAPP_URL` | — | HTTPS-адрес Mini App; при задании появляется кнопка в `/start`. |

## Безопасность

- TG-вход: проверка HMAC-подписи + членство в админах группы
- Сессия: cookie (`HttpOnly`, `SameSite=Lax`)
- Формы: CSRF-токен
- Вход: троттлинг (8 неудач за 5 минут → пауза)
- Mini App: HMAC-проверка `initData`, короткая Bearer-сессия, повторная проверка админского статуса на сервере
- Панель работает по **HTTP** — при доступе снаружи закройте reverse-proxy с HTTPS (nginx/Caddy); это заодно даёт домен для `/setdomain`
